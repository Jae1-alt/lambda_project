# WAF Telemetry & AI Analysis: Python Resilience & The Billing Gatekeeper

Writing a script that works in a perfect environment is easy; writing a serverless pipeline that survives network blips, AI hallucinations, and billing gatekeepers is the mark of a Senior Cloud Engineer. This phase focused on defensive Python programming, ensuring that the security telemetry pipeline never drops data, even when third-party AI services fail.

## 1. Time-Window Querying via `FilterLogEvents`
AWS WAF logs are massive, dense JSON blobs. Attempting to download and parse entire CloudWatch Log Streams would quickly cause the Lambda function to timeout or run out of memory. 

Instead, the Python script utilizes the `boto3` `logs.filter_log_events()` API. By calculating the current time and subtracting a `LOOKBACK_MINUTES` environment variable (converted to epoch milliseconds), the Lambda asks CloudWatch: *"Only give me the WAF blocks that occurred in the last 10 minutes."* 
This time-window querying is highly cost-effective, keeps Lambda execution times under a second, and prevents the script from re-analyzing historical attacks that have already been processed.

## 2. Separation of Logic and Content (External Prompts)
Hardcoding multi-paragraph SOC (Security Operations Center) prompts inside Python scripts is an enterprise anti-pattern. If a security analyst wants to tweak the AI's instructions to get better threat summaries, they should not have to ask a Python developer to change the code, test it, and redeploy the Lambda.

**The Solution:** The prompt is extracted into a standalone `waf_prompt_template.txt` file and packaged alongside the Python script via Terraform's `archive_file` data source. 
* The Python code reads the file into memory at runtime.
* It uses Python's `.format()` method to inject dynamic variables (like the attacker's IP and URI).
* **The Value:** Prompts can be version-controlled and updated independently of the orchestration logic.

## 3. The Golden Rule of SOAR: Persist First
In security automation, the AI is a luxury (Enrichment), but the workflow is a necessity (Remediation/Logging). 

The most critical architectural decision in the Python handler is the order of operations. The Lambda extracts the WAF telemetry and immediately executes `table.put_item()` to save the raw event to the `waf-events` DynamoDB table **before** it ever attempts to call Amazon Bedrock. 

If the AI service experiences an outage, a timeout, or a billing error, the security data is already safely secured in the database. The pipeline never loses evidence just because the AI had a bad day.

## 4. Graceful Degradation & The Billing Gatekeeper
Cloud services fail, and third-party AI models routed through the AWS Marketplace are subject to strict billing gatekeepers. If an AWS account lacks a verified payment instrument, Bedrock will reject the `InvokeModel` call with an `AccessDeniedException`.

A junior engineer's script would crash here, throwing a massive red Python stack trace and failing the Lambda execution. A Senior Engineer uses `botocore.exceptions.ClientError` to inspect the exact AWS API error code and degrade gracefully.

```python
from botocore.exceptions import ClientError

try:
    response = bedrock.invoke_model(modelId=MODEL_ID, body=json.dumps(body))
    # ... parse and return AI summary ...
except ClientError as e:
    err_code = e.response['Error']['Code']
    err_msg = e.response['Error']['Message']
    
    # Catch the AWS Marketplace Billing Gatekeeper
    if err_code == 'AccessDeniedException' and 'payment' in err_msg.lower():
        print("[WARN] AWS Marketplace Billing Gatekeeper triggered.")
        return "[AI Enrichment Skipped: Account billing verification failed.]"
    else:
        return f"[AI Enrichment Skipped: Bedrock {err_code}]"
except Exception as e:
    return f"[AI Enrichment Skipped: AI service unavailable.]"
```
By catching the specific exception and returning a clean string, the Lambda prints a highly visible, actionable warning to CloudWatch (`[WARN] AWS Marketplace Billing Gatekeeper triggered.`) and successfully returns a `200 OK` status code. The security pipeline remains unbroken.

## 5. API Fragmentation: The Multi-Model Payload Trap
A critical lesson learned during this phase is that the Bedrock `InvokeModel` API is a pass-through; every AI provider dictates their own JSON payload schema. 
* **Anthropic (Claude):** Requires the `messages` array, the `anthropic_version` string, and parses the response via `content[0].text`.
* **Amazon (Titan):** Requires the `inputText` string, the `textGenerationConfig` dictionary, and parses the response via `results[0].outputText`.

If an engineer needs to pivot from a third-party model (blocked by billing gatekeepers) to a first-party AWS model (like Titan), the Python payload builder and response parser must be entirely rewritten. This fragmentation reinforces the value of externalizing configurations and keeping the core orchestration logic modular.
